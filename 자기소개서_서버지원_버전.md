# 자기소개서 서버 지원 버전

안녕하세요. 성능과 안정성 문제를 구조적으로 해결하는 과정에서 보람을 느끼는 신입 개발자 배진후입니다.

학부 시절 웹 서비스를 개발하며 백엔드에 흥미를 갖게 되었고, 동시 접속 환경에서 발생하는 병목과 최적화 문제를 다루면서 서버 개발의 매력을 느꼈습니다. 특히 게임 서버는 단순한 API 기반 요청-응답 구조를 넘어 사용자 간 동기화, 상태 관리, 멀티스레딩 처리 등 복합적인 기술 과제를 해결해야 한다는 점에서 더욱 인상 깊었습니다. 저는 이러한 문제를 분석하고 구조적으로 개선해 나가는 과정에서 큰 성취감을 느꼈고, 기능 구현에 그치지 않고 효율성과 안정성을 함께 고려해야 하는 서버 개발이 저와 가장 잘 맞는 분야라고 생각합니다.

포트폴리오에서 가장 중심이 되는 프로젝트는 C++ 기반 게임 서버 프레임워크를 직접 설계하고 구현한 경험입니다. epoll과 io_uring의 차이를 실제 코드와 벤치마크로 검증했고, 여러 차례 재설계를 거치며 성능과 안정성을 개선했습니다.실제 벤치마킹에서 개선 효과를 수치로 검증했습니다.

또한 DirectX 11 기반 멀티플레이 RPG 클라이언트 프로젝트를 통해 서버와 연동되는 클라이언트 구조를 함께 경험했습니다. 던전 렌더링 구조를 인스턴싱 방식으로 설계해 Draw Call 비용을 줄인 경험과 이 과정에서 서버와 클라이언트가 실제로 어떻게 맞물려 동작하는지 전체 흐름을 이해할 수 있었습니다.

졸업작품으로 진행한 팀 프로젝트에서는 서비스 성격에 맞춰 기능을 모듈화하고 협업 체계를 구축하는 경험을 쌓았습니다. API Gateway, User Service, Reservation Service 등 서비스별 저장소를 분리해 운영했고, GitHub Actions와 Argo CD를 활용해 코드 반영부터 배포까지 이어지는 자동화 프로세스를 구현하고자 했습니다. 이를 통해 기술적인 완성도뿐 아니라 협업과 커뮤니케이션의 중요성도 함께 배울 수 있었습니다.

저는 서버 개발자로서 성능과 안정성을 함께 고려하는 구조적인 문제 해결 능력을 더 깊이 키워 나가고 싶습니다. 부족한 실무 경험은 빠르게 학습하고 꾸준히 보완해 나가겠으며, 맡은 업무에 책임감을 가지고 성실하게 임하겠습니다. 또한 팀원들과의 협업과 커뮤니케이션을 중요하게 생각하는 개발자로 성장하겠습니다. 감사합니다.
