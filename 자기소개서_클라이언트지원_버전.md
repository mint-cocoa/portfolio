# 자기소개서 클라이언트 지원 버전

안녕하세요. 성능과 사용자 경험을 함께 고려하며 구조적인 개선을 즐기는 신입 개발자 배진후입니다.

학부 시절 웹 서비스를 개발하며 시스템 전반에 대한 관심을 갖게 되었고, 이후 멀티플레이 게임 프로젝트를 진행하면서 사용자가 직접 체감하는 경험을 만드는 클라이언트 개발의 매력에 더욱 깊이 끌리게 되었습니다. 게임 클라이언트는 단순히 화면을 출력하는 역할에 그치지 않고, 렌더링 성능, 입력 반응성, 네트워크 동기화 결과를 자연스럽게 보여주는 표현 방식까지 함께 고민해야 한다고 생각합니다. 저는 이러한 복합적인 요소를 구조적으로 개선해 나가는 과정에서 큰 보람을 느낍니다.

포트폴리오에서 가장 중심이 되는 프로젝트 중 하나는 DirectX 11 기반 멀티플레이 RPG 클라이언트 구현입니다. 이 프로젝트를 통해 렌더링 구조를 직접 설계하고 최적화하는 경험을 쌓았으며, 던전 렌더링을 인스턴싱 방식으로 재구성해 Draw Call을 4,640개에서 8회 수준으로 줄였습니다. 이를 통해 단순히 기능을 구현하는 데서 끝나는 것이 아니라, 병목을 분석하고 구조를 바꿔 실제 성능 개선으로 이어지게 만드는 개발 방식의 중요성을 배웠습니다.

또한 멀티플레이 환경을 구현하면서 서버와 클라이언트가 실제로 어떻게 맞물려 동작하는지 함께 이해할 수 있었습니다. 클라이언트 입장에서 화면과 입력만 다루는 것이 아니라, 네트워크를 통해 전달되는 상태가 어떤 방식으로 표현되고 사용자의 경험에 영향을 주는지까지 고민하면서 각 영역의 역할과 특성을 함께 바라보는 시각을 기를 수 있었습니다. 이러한 경험은 협업 과정에서도 큰 강점이 된다고 생각합니다.

서버 측 역량도 함께 쌓아왔습니다. C++ 기반 게임 서버 프레임워크를 직접 설계하고 구현하며 epoll과 io_uring의 차이를 실제 코드와 벤치마크로 검증했고, 성능과 안정성을 개선하기 위해 여러 차례 재설계를 진행했습니다. 이 경험 덕분에 클라이언트 개발을 하더라도 서버의 구조와 제약을 함께 이해한 상태에서 더 현실적인 판단을 할 수 있다고 생각합니다.

졸업작품으로 진행한 팀 프로젝트에서는 서비스 성격에 맞춰 기능을 모듈화하고 협업 체계를 구축하는 경험도 쌓았습니다. API Gateway, User Service, Reservation Service 등 서비스별 저장소를 분리해 운영했고, GitHub Actions와 Argo CD를 활용해 코드 반영부터 배포까지 이어지는 자동화 프로세스를 구현하고자 했습니다. 이를 통해 기술적인 완성도뿐 아니라 협업과 커뮤니케이션의 중요성도 함께 배울 수 있었습니다.

저는 클라이언트 개발자로서 사용자에게 직접 전달되는 경험의 완성도를 높이기 위해, 구현과 최적화, 그리고 시스템 전반에 대한 이해를 함께 갖춘 개발자로 성장하고 싶습니다. 부족한 실무 경험은 빠르게 학습하고 꾸준히 보완해 나가겠으며, 맡은 업무에 책임감을 가지고 성실하게 임하겠습니다. 감사합니다.
