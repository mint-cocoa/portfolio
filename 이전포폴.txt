저는 학부 졸업 작품을 웹 서버 개발로 시작하면서 백엔드 및 서버 개발의 관심을 가지게 되었습니다. 여러 사용자가 동시에 접속하는 상황에서 부하 테스트를 진행하며 서버의 최적화에 대해 깊은 관심을 갖게 되었고, 자연스럽게 더 높은 성능과 안정성을 요구하는 분야인 게임 서버 개발에도 매력을 느끼게 되었습니다.

특히 여러 서버 개발 분야 중에서도 게임 서버에 가장 관심을 가지게 된 이유는 기본적인 웹 서버는 주로 api 요청을 통해 요청과 응답 형태의 비교적으로 단순한 통신 패턴이지만, 게임 서버는 실시간적으로 사용자 간 상호작용, 동기화, 상태 관리, 지연 시간 최소화 그리고 멀티쓰레딩 활용 과 같은 복잡하고 다양한 기술적인 도전 과제가 있다는 점이 매력적으로 느껴졌습니다. 저는 이러한 복합적인 문제를 해결하는 과정 자체에서 흥미를 느꼈고, 단순히 기능을 구현하는 수준을 넘어 효율성과 안정성을 지속적으로 관리해야 한다라는 점이 서버 개발이 저에게 가장 잘 맞는 분야라고 판단했습니다.

이러한 관심을 바탕으로 소켓 프로그래밍을 더 깊이 이해하고자 프로젝트를 진행하였고, Linux 환경의 I/O 모델인 epoll과 io_uring을 활용하여 에코 서버를 구현해보았습니다.

이 프로젝트를 통해 TCP 프로토콜의 구조와 클라이언트와의 연결 방식, 세션 관리 등 서버 환경의 기본 개념을 직접 구축하고 이해할 수 있었으며, epoll과 io_uring 각각의 모델로 구현된 서버의 성능을 측정해 비교 분석하기도 했습니다. 어떤 I/O 모델이 더 효율적인지, 각각의 장단점은 무엇인지에 대해 알 수도 있었습니다.

예를 들어, 한 명의 사용자가 입력한 조작이 다른 사용자에게도 거의 동시에 반영되어야 하고, 수많은 클라이언트의 상태를 서버가 정확하고 빠르게 파악하여 일관된 게임 상태를 유지해야 합니다. 이러한 기술적 제약 속에서도 높은 안정성과 확장성을 유지해야 하며, 네트워크 병목이나 세션 관리, 부하 분산 등 다방면의 문제 해결이 요구됩니다. 저는 이러한 복합적인 문제를 해결하는 과정에서 깊은 몰입과 흥미를 느꼈고, 단순히 기능을 구현하는 수준을 넘어 **‘어떻게 더 효율적으로, 더 많은 사용자를 수용하면서 안정적으로 서비스할 수 있을까’**라는 고민을 지속할 수 있다는 점에서 게임 서버 개발이 저에게 가장 잘 맞는 분야라는 확신을 가지게 되었습니다.

안녕하세요, 네트워크 프로그래밍과 서버 개발 분야에 깊은 관심을 가진 소프트웨어 전공 신입 지원자입니다. 규모가 매우 큰 게임 서비스인 메이플스토리의 안정적인 운영과 효율성을 관리하는 것에 매우 흥미를 느껴 지원하게 되었습니다.

포트폴리오에서는 리눅스 환경의 효율적인 I/O 처리 모델에 대한 연구를 기반으로, epoll과 io_uring과 같은 비동기 I/O 모델의 성능 차이를 비교 분석한 프로젝트를 진행했습니다. 이를 위해 직접 소켓 생성 및 래핑 클래스 개발, 메시지 전송 및 동시 접속 처리 구조 구현 등을 통해 실제 네트워크 서버를 구축하고, 성능 테스트를 반복하면서 실무에 가까운 경험을 쌓았습니다.

고성능 서버 구축을 목표로, 멀티쓰레딩을 활용하여 CPU 코어마다 독립적인 I/O 처리 인스턴스를 배치하고 이를 효율적으로 관리하기 위한 매니저 클래스를 개발했습니다. 성능 분석을 위해 오픈소스 벤치마킹 도구를 수정하여 비교 평가를 진행하였으며, 이 결과는 개인 웹사이트를 통해 문서화하여 포트폴리오로 구성했습니다. 

또한, 학부생 졸업 작품으로는 클라우드 네이티브 환경에 대한 경험을 쌓기 위해 Kubernetes와 Docker를 활용한 컨테이너 기반의 서비스 구축 실습을 진행하였고, CI/CD 파이프라인 구축을 위해 argocd를 연동하여 자동화 배포 환경을 구성해 보았습니다. 이를 통해 실시간 장애 대응과 효율적인 릴리즈 관리의 중요성을 체감하였고, 이를 실무에 적용할 수 있는 기반을 마련했습니다.

데이터베이스 측면에서는 MongoDB를 이용하여 사용자 정보를 관리하고 게시글, 댓글 등의 데이터를 처리하는 CRUD 기반의 웹 서비스를 직접 구현한 경험이 있습니다. 해당 프로젝트에서는 데이터 스키마 설계부터 API 라우팅, 예외 처리 및 응답 구조 설계까지 전반적인 백엔드 로직을 설계하였으며, 이를 Python 기반의 FastAPI 프레임워크를 통해 RESTful API 서버 형태로 구축했습니다. 또한, 비동기 요청 처리와 함께 JWT를 활용한 인증 시스템을 도입하여 실제 서비스 환경에 가까운 구조를 구현해 본 경험이 있습니다.

신입 개발자로서 아직은 배워야 할 것이 많지만, 새로운 기술에 대한 열정과 문제 해결에 대한 집요함을 가지고 빠르게 성장해 나갈 자신이 있습니다. 메이플시스템개발실에서의 업무를 통해 실시간 서비스 운영의 노하우와 게임 라이브 서비스를 위한 시스템 개발 역량을 쌓으며, 메이플스토리의 안정적인 서비스를 함께 만들어가는 일원으로 성장하고 싶습니다.
